---
title: 'Creating Tickets'
---
<Steps>
  <Step title="Navigate to your event's overview page and open the '③ Tickets' tab">
    <img width=541 height=700 src="/images/creating-campaign/ticketing/creating tickets/3 Tix Location.png"/>
  </Step>
  <Step title="Click 'Add Ticket'">
    <img src="/images/creating-campaign/ticketing/creating tickets/Add Tix Button Location.png"/>
  </Step>
  <Step title="Add ticket name/tier">
    Enter the name or category (GA, VIP, Concession, etc) for this ticket type. Additional ticket types can always be added later.
    <img src="/images/creating-campaign/ticketing/creating tickets/Tix Name Field.png"/>
    <Warning>
        Please note that future updates made to the "Ticket Name" field may not automatically appear on NFT 
        marketplaces for NFTs that have already been minted. To ensure the latest metadata is displayed, 
        manually refresh the NFT metadata on the respective marketplaces.
    </Warning>
    <Frame caption="Where customers will see the different ticket types">
        <img src="/images/creating-campaign/ticketing/creating tickets/Tix Type Location.png"/>
    </Frame>
  </Step>
  <Step title="Add ticket image and description">
    Upload an image and a brief description for this ticket type.
    <img src="/images/creating-campaign/ticketing/creating tickets/Add Tix Img and Desc.png"/>
    <Warning>
        Please note that future updates made to ticket images may not automatically appear on NFT 
        marketplaces for NFTs that have already been minted. To ensure the latest metadata is displayed,
        manually refresh the NFT metadata on the respective marketplaces.
    </Warning>
    <Tabs>
        <Tab title="Ticket Description">
            <Frame caption="Where customers will see the ticket description during checkout">
                <img src="/images/creating-campaign/ticketing/creating tickets/Tix Desc Location.png"/>
            </Frame>
        </Tab>
        <Tab title="Ticket Image">
            <Frame caption="Where customers will see the ticket image during checkout">
                <img src="/images/creating-campaign/ticketing/creating tickets/Tix Img Checkout.png"/>
            </Frame>

            <Frame caption="Where customers will see the ticket image in their ticket confirmation email">
                <img src="/images/creating-campaign/ticketing/creating tickets/Tix Img Email.png"/>
            </Frame>
        </Tab>
    </Tabs>
  </Step>
  <Step title="Specify ticket price and sales period">
    Enter the desired price for this ticket type, as well as when sales of this ticket type will begin and end.
    <img src="/images/creating-campaign/ticketing/creating tickets/Tix Price Sales Period.png"/>
    <Tip>
        Set the price to 0 for free tickets.
    </Tip>
    <Tip>
        To change the currency type, return to "① Info" in the sidebar and change the value in the "Currency" field.
    </Tip>
  </Step>
  <Step title="Hidden tickets, gated tickets, limited quantities, invite only tickets, etc (Optional)">
    <Accordion title="Advanced Options">
      The advanced options menu allows you to specify the number of available tickets of this type, the maximum number of tickets
      of this type purchasable in a transaction, gating settings via NFT/email/wallet, whether this ticket type should be hidden
      on the event page, and whether users require a special code to purchase this ticket type.
      <img src= "/images/creating-campaign/ticketing/creating tickets/Advanced Options.png"/>
      <Tabs>
        <Tab title="Hide Tickets">
          <img src="/images/creating-campaign/ticketing/creating tickets/hide.png"/>
          Click the "Hide?" toggle to make this ticket type invisible to users on the event and checkout pages.
        </Tab>
        <Tab title="Limit Quantity">
          <img src="/images/creating-campaign/ticketing/creating tickets/limited quantity.png"/>
          Click the "Limited Quantity" toggle to specify the number of available tickets of this type. When sales of this ticket type meet the specified quantity,
          the ticket will no longer be available for purchase.
        </Tab>
        <Tab title="Limit Purchases per Transaction">
          <img src="/images/creating-campaign/ticketing/creating tickets/limit per transaction.png"/>
          Enter a value in the "Limited Purchase per Transaction" field. Users will not be able to purchase more of this ticket type in a single transaction (for
          themselves and others) than the specified value.
        </Tab>
        <Tab title="Limit Purchases per User">
          <img src="/images/creating-campaign/ticketing/creating tickets/limit per user.png"/>
          Click the "Limited Purchase per user" toggle to activate this option, then enter a value in the "Maximum Purchase per user" field. Users will not
          be able to purchase more of this ticket type for themselves than the specified value.

          For example, if the value was set to 1, the user could not purchase more than one of this ticket type for themselves, but they could still purchase
          tickets on the behalf of others.
        </Tab>
      </Tabs>
      <Info>
          To gate tickets by NFT ownership or Email/Wallet addresses, please refer to this [guide](/creating-campaign/ticketing/gating).
      </Info>
      <Info>
          To require users to enter a special code to unlock tickets, please refer to this [guide](/creating-campaign/ticketing/inviteonly).
      </Info>
      When you are ready, click "Save" to close the ticket details window, then click "Next."
    </Accordion>
  </Step>
  <Step title="Add questions (Optional)">
    <Accordion title="Adding Questions">
      <img src="/images/creating-campaign/ticketing/creating tickets/Add Qs.gif"/>
      If applicable, add questions to collect information from users who are purchasing tickets. These questions 
      will be shown to users during checkout.
      <img src="/images/creating-campaign/ticketing/creating tickets/Qs.png"/>
      Responses can be viewed from the event's Sales tab, located in the event dashboard.
      <img src="/images/creating-campaign/ticketing/creating tickets/Reponses.gif"/>
    </Accordion>
  </Step>
  <Step title="Specify contract settings and payment options">
    Choose a collection name and symbol for **all ticket types associated with your event.** The collection name will be used as the collection title when tickets are listed on marketplaces.
    <img src="/images/creating-campaign/ticketing/creating tickets/Contract Blank.png"/>
    <Warning>
        Collection name and symbol cannot be changed after your event is published.
    </Warning>
    <img src="/images/creating-campaign/ticketing/creating tickets/Collection Name OS Location.png"/>
    Select a blockchain to mint your event and tickets on, as well as a contract type.
    <img src="/images/creating-campaign/ticketing/creating tickets/Contract Filled.png"/>
    <Info>
        For more information on selecting a blockchain and contract type, please consult section "7. Technical Information."
    </Info>
    Select a payment option if applicable. Moongate supports both credit cards and crypto options. **Please contact us if you will be collecting payment for your event.**
    <img src="/images/creating-campaign/ticketing/creating tickets/Payment Options.png"/>
  </Step>
  <Step title="Publish your event">
    Once you have finalized your event and ticket details, click "Publish" to launch your event.
    <img src="/images/creating-campaign/ticketing/creating tickets/Publish Button Location.png"/>
  </Step>
</Steps>
